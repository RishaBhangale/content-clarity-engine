Standard Operating Procedure: Data Handling
Version: 4.2
Last Updated: March 2025
Department: IT Operations

Section 1: Purpose
This SOP outlines the procedures for handling, storing, and disposing of data within the organization's IT infrastructure.

Section 2: Data Classification
The organization uses four data classification levels: Public, Internal, Confidential, and Restricted. The Restricted category was added in Q2 2024 for highly sensitive data requiring additional security controls.

Section 3: Data Storage
3.1 Storage Requirements
All data must be stored in approved cloud storage systems. On-premise storage is prohibited for any data classified as Confidential or Restricted.

3.2 Encryption
Data at rest must be encrypted using AES-256. Data in transit must use TLS 1.3 or higher.

Section 4: Data Retention
4.1 Customer Data
Customer data shall be retained for a minimum of 1 year following account termination to comply with regulatory requirements and potential legal holds.

4.2 Operational Logs
System and application logs are retained for 90 days. Security audit logs are retained for 2 years.

Section 5: Data Deletion
5.1 Deletion Procedures
Data deletion requests must be submitted through the ServiceDesk portal. Manager approval is required within 48 business hours before proceeding with deletion.

5.2 Verification
All deletions must be verified by a second team member and documented in the deletion log.

Section 6: Security Reviews
Annual security reviews are conducted by the compliance team and results shared during the yearly audit meeting. The review covers all data handling procedures and access controls.

Section 7: Password Requirements
Passwords must be a minimum of 14 characters and include uppercase, lowercase, numbers, and special characters. Passwords expire every 90 days and cannot be reused for 12 cycles.

Section 8: Deployment Process
All deployment requests must be submitted through the ServiceDesk portal. Manager approval is required within 48 business hours before proceeding.

Section 9: Access Control
Access to production data requires VP-level approval. All access is logged and reviewed quarterly. Access grants expire after 90 days and must be renewed.
